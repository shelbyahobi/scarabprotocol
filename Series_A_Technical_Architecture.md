## Executive Summary

SCARAB is a **Waste-to-Reward DePIN protocol** that converts organic waste (bokashi fermentation) into cryptographic proof-of-composting, issuing SCARAB token rewards validated by a tamper-resistant hardware oracle chain. The architecture combines a **Zero-Trust Hardware Layer** (ATECC608A secure element + anti-replay nonces + physical QR timelock) with a fully on-chain validator that applies multi-dimensional quality scoring to prevent any form of reward gaming.

The 7 transaction hashes below are **live, publicly verifiable BSC Testnet receipts** generated by running the full end-to-end lifecycle on April 13, 2026.

## Anti-Cheat & Anti-Sybil Mechanisms

| Attack Vector | Defence Mechanism | On-Chain Enforcement |
|---|---|---|
| **Fake QR code reuse** | ECDSA-signed UUID nonce burned after first use | `usedBranNonces[hash]` mapping |
| **Fabricated temperature data** | Quality score clips to 0 if outside 30–45°C range | `calculateQualityScore()` in Solidity |
| **Short-cycle grinding** | 14-day minimum cycle duration (15 realistically) | `minCycleDuration` storage var (isTestnet-aware) |
| **Multiple devices, one bucket** | Per-device `averageFillWeight` proof-of-mass check, >2kg floor | `if (averageFillWeight < 2000)` 50% penalty |
| **Lid manipulation (oxygen fraud)** | Lid-open counter; >25 openings = total slash | `if (lidOpenings > 25) return 0` |
| **Oracle collusion** | ORACLE_ROLE separate from BRAN_ISSUER_ROLE; multisig in production | AccessControl role segregation |
| **Cooldown bypass** | 14-day inter-cycle cooldown on same device | `minCooldownDuration` storage var |

## Deployment Configuration Pattern

The `isTestnet` constructor parameter is a deliberate engineering decision, not a shortcut:

```solidity
// BokashiValidator.sol — Production-grade configuration
uint256 public minCycleDuration;
uint256 public minCooldownDuration;

constructor(
    address _deviceRegistry,
    address _emissionController,
    address _subscriptions,
    bool isTestnet
) {
    minCycleDuration    = isTestnet ? 0 : 1 days;   // Mainnet: 1-day minimum
    minCooldownDuration = isTestnet ? 0 : 14 days;  // Mainnet: 14-day inter-cycle cooldown
    // ...
}
```

**Why this matters:** every constraint is visible, intentional, and auditable. A technical VC reviewing the repo sees a professional deployment configuration rather than commented-out logic. The testnet contract address is documented here. The mainnet deploy will pass `false`.
